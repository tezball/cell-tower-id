# **Advanced Analytics in Mobile Network Security: IMSI Catcher Detection, Heuristic Modeling, and the Mitigation of False Positives in Urban Topographies**

The proliferation of rogue cellular base stations—commonly referred to as International Mobile Subscriber Identity (IMSI) catchers, "Stingrays," or Cell-Site Simulators (CSS)—represents one of the most pervasive threats to telecommunications privacy. These devices exploit foundational vulnerabilities in cellular network architectures to covertly intercept traffic, track physical locations, and extract unencrypted identifiers from mobile devices. Concurrently, the defensive landscape has evolved, relying on crowdsourced data and heuristic anomaly detection algorithms operating within the user space of mobile operating systems.

However, detecting an active cyberattack from the vantage point of a mobile handset is fraught with complexity. As cellular infrastructure rapidly modernizes through network densification, small cell deployments, and Multi-Operator Core Network (MOCN) sharing agreements, the legitimate behaviors of urban networks increasingly mimic the cryptographic and spatial signatures of an active IMSI catcher attack.

This comprehensive report is designed to bridge the knowledge gap between foundational radio frequency principles and advanced algorithmic detection strategies. By dissecting the underlying architecture of cellular telecommunications, analyzing the mechanics of cell-site simulators, and exhaustively evaluating an 11-point heuristic detection model against the backdrop of real-world network dynamics in regions such as Dublin, Ireland, this document provides a rigorous framework for identifying true cryptographic anomalies while systematically eliminating false positives.

## **1\. Foundational Primer: Cellular Network Architecture and Terminology**

To comprehend the mechanisms by which a rogue device compromises a mobile network, one must first master the architectural baseline of legitimate telecommunications. Cellular networks operate on a hierarchy of geographical subdivisions, cryptographic identifiers, and radio frequency channels that orchestrate the seamless transfer of data as a user moves through the physical world.

### **1.1 Core Telecommunications Lexicon**

The following structural identifiers are globally standardized by the International Telecommunication Union (ITU) and the 3rd Generation Partnership Project (3GPP) to manage subscribers and route traffic efficiently.1

| Identifier | Full Name | Functional Description and Structural Architecture |
| :---- | :---- | :---- |
| **MCC** | Mobile Country Code | A standardized 3-digit number assigned by the ITU to identify a specific country or geographic region. For example, the MCC for the Republic of Ireland is 272, the United Kingdom is 234, and the United States occupies the 310–316 block.1 |
| **MNC** | Mobile Network Code | A 2- or 3-digit number identifying a specific network operator within an MCC domain. In Ireland, the MNC for Vodafone is 01, Three is 05 (and legacy O2 is 02), and Eir is 03\.1 |
| **PLMN** | Public Land Mobile Network | The concatenated string of the MCC and MNC. The PLMN uniquely identifies a single carrier's domain on the global stage.2 |
| **IMSI** | International Mobile Subscriber Identity | A permanent, globally unique 15-digit identifier physically embedded within a SIM card. It consists of the MCC, the MNC, and the Mobile Subscriber Identification Number (MSIN). Protecting the IMSI from over-the-air interception is the primary goal of modern cellular privacy architectures.1 |
| **TMSI** | Temporary Mobile Subscriber Identity | A randomized, rotating identifier assigned by the core network's Mobility Management Entity (MME). To prevent tracking, the network uses the TMSI for routing rather than the permanent IMSI.6 |
| **LAC / TAC** | Location Area Code / Tracking Area Code | A logical grouping of multiple cell towers. As a mobile device crosses the geographical boundary from one TAC to another, it must initiate a Tracking Area Update (TAU) to inform the core network of its new location.5 |
| **CID / ECI** | Cell Identity / EUTRAN Cell ID | A unique serial number assigned to a specific hardware antenna sector on a cellular tower.12 |
| **eNodeB / gNB** | Evolved Node B / Next Gen Node B | The physical radio base station hardware in 4G LTE (eNodeB) and 5G NR (gNB) networks. A single eNodeB typically manages multiple localized sectors (CIDs).4 |
| **EARFCN** | E-UTRA Absolute Radio Frequency Channel Number | An integer indicating the specific carrier frequency band used for transmission.4 |
| **PCI** | Physical Cell Identity | A crucial, short-hand radio-layer identifier (ranging from 0–503 in LTE and 0–1007 in 5G) used by a mobile device to separate overlapping radio signals during spatial synchronization.12 |

### **1.2 The Physics of Cell Attachment and Signal Measurement**

When User Equipment (UE)—such as an Android smartphone—is powered on, its baseband processor immediately begins scanning the radio environment. The UE is programmed to seek out the strongest, highest-quality signal that belongs to its home operator (identified by the PLMN) or an authorized roaming partner.

The primary metric utilized to evaluate this connection in LTE and 5G networks is the Reference Signal Received Power (RSRP). Measured in decibels relative to a milliwatt (dBm), RSRP quantifies the power of the specific reference signals spread across the assigned frequency bandwidth.1 Because RSRP operates on a logarithmic scale, an RSRP of \-75 dBm indicates an exceptionally strong, saturated signal (likely within close physical proximity to the tower), whereas an RSRP of \-115 dBm indicates severe signal degradation at the very edge of the cell's coverage area.17

Furthermore, cellular networks utilize Time Division Multiple Access (TDMA) principles. Data is transmitted in highly precise time slots. Because radio waves travel at the speed of light, a UE positioned 10 kilometers away from a tower experiences a longer propagation delay than a UE located 100 meters away. To prevent the delayed signals of distant devices from colliding with the signals of closer devices, the network calculates a Timing Advance (TA) value.18

The TA acts as a negative offset, instructing the UE to transmit its uplink data fractionally early so that it arrives at the eNodeB in perfect synchronization. In LTE networks, the TA value (![][image1]) is quantized based on a fundamental time unit ![][image2], where one TA step equates to approximately 0.52 microseconds.19 This temporal measurement translates directly to physical space: a single TA step corresponds to roughly 78 meters of radial distance from the antenna.20 Therefore, if a UE reports a TA of 0 or 1, it is deterministically established that the device is within approximately 150 to 550 meters of the physical base station.18

## **2\. The Threat Landscape: Mechanics of IMSI Catcher Attacks**

IMSI catchers, colloquially known by brand names such as Stingray or Hailstorm, were originally developed for military and intelligence applications but have increasingly proliferated into the hands of local law enforcement, corporate espionage actors, and organized criminal syndicates. These devices operate by manipulating the fundamental trust protocols established in early cellular standards.

### **2.1 The Trust Deficit and Active Protocol Downgrades**

The foundational vulnerability of global telecommunications lies in the architecture of 2G (GSM) networks. In the GSM standard, the network cryptographically authenticates the mobile device to ensure the subscriber is authorized to use the service and generate billing records. However, the mobile device is not required to authenticate the network.10 A rogue base station can simply broadcast the correct PLMN, and the UE will blindly trust it.

Once connected, an active IMSI catcher operating in 2G mode exploits this lack of mutual authentication by issuing a command to the UE to completely disable over-the-air encryption (utilizing the A5/0 algorithm) or downgrade to an inherently flawed encryption standard (A5/1 or A5/2).10 This renders all voice calls, SMS messages, and data traffic in plaintext, enabling complete Man-In-The-Middle (MITM) interception.10

Modern 3G (UMTS), 4G (LTE), and 5G (NR) networks were specifically designed to eliminate this vulnerability by enforcing mutual authentication—the UE and the network must mathematically prove their identities to one another.10 However, backward compatibility remains a fatal flaw. To support legacy devices and ensure coverage in rural areas, modern networks allow UEs to seamlessly fall back to 2G or 3G.22

Sophisticated IMSI catchers exploit this by engaging in "bidding down" or downgrade attacks.22 While 4G requires mutual authentication, the initial handshake messages used to negotiate network capabilities—such as the Radio Resource Control (RRC) broadcasts and Tracking Area Update (TAU) requests—are transmitted before the cryptographic keys are established.22 An attacker can establish a fake 4G eNodeB, wait for the target UE to attempt an attachment, and then respond with an unauthenticated TAU Reject message containing a specific error code (e.g., "Network Failure" or "LTE Services Not Allowed").22 The UE, believing the local 4G network is broken, dutifully drops its connection and searches for a 2G network, which the attacker is simultaneously broadcasting.22

### **2.2 Passive Interception, Identity Harvesting, and Silent Paging**

Not all IMSI catchers operate as active MITM relays. Passive or semi-passive catchers focus entirely on surveillance, identity harvesting, and location tracking.10

To protect privacy, networks assign a rotating Temporary Mobile Subscriber Identity (TMSI). However, if a UE wanders into a new network area and the local core network databases fail, the network must have a mechanism to recover the connection.6 When a UE connects to a rogue base station broadcasting a spoofed Location Area Code (LAC), the UE transmits its TMSI.6 The rogue station replies with an "Identity Request" message, claiming the TMSI is invalid or unrecognized.6 Bound by protocol compliance, the UE is forced to transmit its permanent IMSI in plaintext.6 The IMSI catcher records the identity, drops the connection, and allows the UE to return to the legitimate network—an interaction that occurs in milliseconds without user notification.10

Furthermore, attackers deploy localized tracking techniques such as the "Silent SMS" (Short Message Type 0\) or the silent call.6 A Type 0 SMS is uniquely defined in the GSM 03.40 standard.25 When transmitted to a target device, the baseband processor of the UE acknowledges the receipt of the message, generating a burst of radio traffic.25 However, the mobile operating system immediately discards the message without triggering a visual notification, vibration, or audio alert.25 By repeatedly sending silent messages, an adversary operating a directional antenna or a localized IMSI catcher can actively triangulate the physical location of the device based on the radio acknowledgments.25

## **3\. The Android Operating System and Detection Constraints**

To counteract these covert surveillance mechanisms, developers have built heuristic detection engines. However, developing these engines requires navigating the strict hardware and software abstractions of modern smartphone architecture.

A mobile device is driven by two distinct computational environments: the Application Processor, which runs the Android Operating System, and the Baseband Processor (modem), which runs a proprietary real-time operating system responsible for all radio communications.21 These two processors communicate via the Radio Interface Layer (RIL).21

### **3.1 The Limits of the TelephonyManager API**

For security and stability, Android abstracts the underlying baseband complexities. Application developers must query the radio state using public APIs, primarily the TelephonyManager.getAllCellInfo() method.28 This method returns standardized objects (e.g., CellInfoLte, CellInfoGsm) that encapsulate basic network parameters: the PLMN, the CID, the TAC, the PCI, the RSRP, and the Timing Advance.28

This architectural abstraction is the fundamental limitation of user-space IMSI catcher detection.6 A non-root application cannot inspect the raw Layer 3 Non-Access Stratum (NAS) messages.6 Therefore, the application cannot directly observe the malicious "TAU Reject" message causing a downgrade, nor can it explicitly read the "Identity Request" extracting the IMSI.6

Instead, detection applications must operate on a model of heuristic inference. They cannot see the attack itself; they can only observe the macroscopic physical "ripples" the attack creates in the local radio environment—sudden changes in network generation, geographical impossibilities, signal volatility, and configuration inconsistencies.30

Research frameworks evaluating detection apps, such as the "White-Stingray" study, have demonstrated that while basic heuristics effectively catch poorly configured simulators, highly sophisticated attackers who perfectly mimic local network parameters can evade rudimentary detection.6 Consequently, the detection logic must be exhaustively hardened against complex scenarios to maintain efficacy.

## **4\. Exhaustive Diagnostic Analysis of Heuristic Detectors**

The detection framework currently under review operates via a point-based severity model utilizing 11 distinct algorithmic triggers. By aggregating these alerts, the application classifies a mobile session as Normal, Suspicious, or Alert. To optimize this engine and eliminate false positives, each heuristic must be subjected to rigorous technical scrutiny.

### **4.1 Geo-Velocity and Spatial Chronology Anomalies**

**IMPOSSIBLE\_MOVE (High Severity)**

* **Trigger Mechanism:** Evaluates the spatial chronology of a known Cell Identity (MCC/MNC/TAC/CID). If a tower previously mapped by the user is suddenly observed at a geographic distance exceeding 20 kilometers from its historic baseline, the alert fires. In LTE environments, the algorithm executes a safety check against sibling sectors (CIDs sharing the same eNodeB) to verify the anomaly.  
* **Attack Vector:** Cell-site simulators are highly mobile, often deployed in surveillance vans, low-flying aircraft, or carried in backpacks.30 If an attacker spoofing a known Dublin cell tower drives to Galway and activates the equipment, the UE will suddenly detect a "Dublin" tower 200 km away.30 This utilizes the same "impossible travel" algorithms prevalent in enterprise identity management.34  
* **False Positive Origins & Mitigation:** The primary vector for false positives is a corrupted GPS baseline. If the user's initial observation of a tower occurred during a GPS multipath error (e.g., reflecting off skyscrapers) or a "cold start" lacking satellite lock, the tower's geographic anchor is stored incorrectly. When a highly accurate GPS fix is subsequently acquired, the static tower appears to have teleported. Furthermore, legacy GSM (900 MHz) signals possess exceptional propagation characteristics, capable of traveling dozens of kilometers over open water or through atmospheric ducting. Because GSM lacks the eNodeB sibling-sector fallback structure 13, long-range GSM propagation can trigger spurious impossible move alerts.

**TRANSIENT\_TOWER (Medium Severity)**

* **Trigger Mechanism:** Identifies extreme chronological brevity. Fires if a tower appears and vanishes in under 5 minutes while the UE is stationary, or under 20 minutes if the UE is driving (speed \> 3.0 m/s).  
* **Attack Vector:** Portable simulators are utilized for targeted, surgical extractions. They are powered on, the target's identity is harvested, and the device is immediately deactivated to evade counter-surveillance.30  
* **False Positive Origins & Mitigation:** The most persistent false positive occurs during vehicular transit. Passing a highway-adjacent microcell will naturally produce a transient chronological signature.37 The application's integration of a driving gate (extending the window to 20 minutes if speed \> 3.0 m/s) mitigates, but cannot entirely eliminate, the natural transience of drive-by telecommunications.

### **4.2 Cryptographic Downgrades and Routing Fluctuations**

**DOWNGRADE\_2G (High Severity) & DOWNGRADE\_3G (Medium Severity)**

* **Trigger Mechanism:** Monitors the active Radio Access Technology (RAT). If a UE connected to a secure LTE or 5G NR network abruptly drops to an insecure GSM (2G) or WCDMA/CDMA (3G) connection while maintaining the exact same operator PLMN (MCC+MNC), the alert triggers.1  
* **Attack Vector:** This is the absolute hallmark of an active bidding-down attack. The simulator generates localized noise or issues unauthenticated signaling messages to break the LTE connection, forcing the UE into the vulnerable legacy environment where encryption can be stripped.10  
* **False Positive Origins & Mitigation:** The heuristic design includes a critical "same-operator gate." If the MCC or MNC changes during the downgrade, the alert is suppressed, correctly interpreting the event as legitimate roaming (e.g., dropping from a primary 4G network into a partner's 2G network for rural coverage).2 However, legitimate hardware failures at the physical eNodeB, or highly specific topographical dead zones where high-frequency LTE bands fail to penetrate but lower-frequency GSM bands succeed, represent residual false-positive risks.

**LAC\_TAC\_CHANGE (Medium Severity)**

* **Trigger Mechanism:** Monitors the Location Area Code (2G/3G) or Tracking Area Code (LTE/5G) for sudden changes within the same operator network. While stationary, it fires on the first flip; while driving (\> 3.0 m/s), it requires ≥ 3 consecutive flips within a 60-second window to trigger.5  
* **Attack Vector:** To force a UE to execute a registration sequence and transmit its IMSI, catchers intentionally broadcast a LAC/TAC that differs from the surrounding ambient network.6  
* **False Positive Origins & Mitigation:** Dense urban topographies naturally trigger handovers. As a pedestrian moves through overlapping sector boundaries, multipath reflections off buildings can cause the UE to "ping-pong" rapidly between two distinct eNodeBs.38 If these eNodeBs sit on the administrative boundary of two distinct Tracking Areas, natural network optimization will generate 3+ TAC flips within 60 seconds, falsely mimicking an attack. The driving gate suppression is crucial here to prevent an avalanche of alerts during highway transit through multiple rural administrative zones.

### **4.3 Network Identity Spoofing and Saturation**

**OPERATOR\_MISMATCH (High Severity)**

* **Trigger Mechanism:** Cross-references the broadcast PLMN against geographic expectations. Triggers specifically when the MCC falls within the United States range (310–316), but the concatenated MCC/MNC pair is absent from a hardcoded whitelist of known, legitimate carriers.  
* **Attack Vector:** Inexperienced operators of cell-site simulators may fail to accurately configure their equipment, leaving default network codes, experimental test PLMNs (e.g., MCC 001), or inaccurate regional codes active during deployment.30  
* **False Positive Origins & Mitigation:** The telecommunications industry undergoes constant corporate restructuring. The proliferation of localized Mobile Virtual Network Operators (MVNOs) guarantees that any static, hardcoded whitelist will eventually become obsolete. If a legitimate regional carrier begins operations and is not immediately added to the application's local database, a high-severity alert will trigger.

**SIGNAL\_ANOMALY (Medium/Low Severity)**

* **Trigger Mechanism:** Analyzes RSRP saturation against historical data. Fires if the current RSRP spikes 20–35 dBm (Low) or \> 35 dBm (Medium/High) above the rolling local average for that specific operator, provided a minimum baseline of 5 measurements exists. Suppressed entirely during vehicular transit (\> 3.0 m/s).  
* **Attack Vector:** The most rudimentary method for capturing a target UE is absolute signal dominance. By radiating at a power level substantially higher than the ambient macro network, the simulator guarantees the UE will select it during the cell reselection process.10  
* **False Positive Origins & Mitigation:** Microcell and femtocell deployments inherently cause sudden spikes in RSRP. A pedestrian walking directly beneath a picocell mounted on a streetlamp will experience a massive, immediate RSRP spike.39 While the driving gate suppresses drive-by anomalies, pedestrian proximity to legitimate infrastructure remains a persistent source of Low-severity residual alerts.

### **4.4 Spatial Proximity and Attenuation Volatility**

**SUSPICIOUS\_PROXIMITY (High Severity)**

* **Trigger Mechanism:** A complex multi-variate detector relying on the physics of signal propagation. It triggers if the Timing Advance is ![][image3] (indicating a radial distance ![][image4] meters) 18, but the RSRP is inexplicably moderate (-95 to \-85 dBm) rather than saturated (![][image5] dBm). This requires the UE to be stationary or moving at pedestrian speeds (![][image6] m/s), subject to a 30-second hysteresis delay after stopping, and requires that the UE has never previously observed this cell at a saturated signal level.  
* **Attack Vector:** IMSI catchers deployed covertly—such as those hidden in briefcases or parked vehicles near a target—must limit their transmission power to avoid detection by broader geographic sensors.41 A low Timing Advance betrays their immediate physical proximity, but their artificially lowered transmission power creates a paradox: the signal is perfectly synchronized in time (close) but highly attenuated in power (distant).18  
* **False Positive Origins & Mitigation:** This heuristic is exquisitely sensitive to biological signal attenuation. The 30-second speed hysteresis brilliantly mitigates vehicular "brake-light dips," where a driver stops at a red light next to a tower, triggering a stationary proximity event. However, human body shadowing introduces extreme volatility.43 Extensive research utilizing Ultra-Wideband (UWB) and microwave pedestrian tracking demonstrates that the human torso can introduce 10 to 20 dB of signal attenuation depending on the angle of incidence.43 A mobile device stored in a back pocket, oriented away from a legitimate small cell located just 40 meters away (TA \= 0), may report a highly degraded RSRP of \-90 dBm due entirely to biological mass and indoor multipath fading.38 This physical reality directly mirrors the signature of a covert Stingray, falsely triggering the SUSPICIOUS\_PROXIMITY alert.

### **4.5 Configuration Inconsistencies and the PCI Dilemma**

**PCI\_COLLISION (High Severity) & PCI\_INSTABILITY (Medium Severity)**

* **Trigger Mechanism:** PCI\_COLLISION fires if, within a 2 km radius over 7 days, the identical Physical Cell Identity is broadcast by multiple distinct eNodeBs (LTE) or CIDs (GSM). PCI\_INSTABILITY fires if a previously cached Cell Identity begins broadcasting a different PCI. Both are suppressed while driving and re-evaluated every 6 hours.  
* **Attack Vector:** The PCI is a fundamental 9-bit identifier.12 Rogue base stations frequently parrot the PCI of adjacent legitimate towers to orchestrate overshadowing attacks, bypass neighbor list verification, or induce artificial handovers.31 Poorly configured simulators may also broadcast random PCIs that clash violently with local topological planning.17  
* **False Positive Origins & Mitigation:** The use of PCI as a heuristic discriminator is severely challenged by modern network densification strategies. The suppression of sibling sectors (multiple antennas on the same eNodeB sharing a PCI for Carrier Aggregation) mitigates intra-site false positives.12 However, inter-site PCI reuse is an unavoidable mathematical necessity. This phenomenon drives the highest volume of false positives in urban environments and requires dedicated analysis.

## **5\. Network Densification, HetNets, and PCI Collision Mathematics**

To accurately refine the PCI\_COLLISION heuristic, it is necessary to contextualize the rules engine against real-world cellular deployments. The network topographies of Dublin, Ireland—specifically contrasting suburban corridors like Lucan with ultra-dense environments like the Smart Docklands—provide a perfect framework for understanding why PCI logic fails.

### **5.1 The Finite Identifier Pool and Graph Coloring Optimization**

Unlike the permanent 15-digit IMSI or the 28-bit Cell Identity, the Physical Cell Identity is a highly restricted, localized shorthand.12 In LTE, there are exactly 504 available PCIs.12 This number is derived mathematically by combining 168 Secondary Synchronization Signal (SSS) groups with 3 Primary Synchronization Sequences (PSS), calculated as ![][image7].12 In 5G NR networks, this pool is expanded to 1008\.16

Because the pool is severely limited, national network operators must continuously reuse the same PCIs.16 The fundamental objective of Radio Frequency planning is to maximize the "reuse distance"—the geographic separation between two towers assigned the identical PCI.15 If two adjacent cells share a PCI, the UE suffers from "PCI Confusion" or "PCI Collision," resulting in catastrophic interference and handover failures.15

Optimizing PCI distribution is modeled mathematically as a graph coloring problem, where cells represent vertices, overlapping coverage areas represent edges, and PCIs represent colors.16 Advanced self-organizing networks (SON) employ algorithms such as Maximum Degree First coloring, Integer Linear Programming (ILP), or Biased Random-Key Genetic Algorithms (BRKG) to autonomously distribute identifiers and mitigate conflicts.16

### **5.2 The 2 km Fallacy in Ultra-Dense Urban Topographies**

In suburban environments such as Lucan, network coverage is predominantly supplied by macrocells mounted on high-elevation towers.48 These macrocells project signals over several kilometers.39 In such an environment, an operator can easily maintain a PCI reuse distance of 5 to 10 kilometers.15 Consequently, the app's PCI\_COLLISION rule—which monitors a 2 km radius for identical PCIs—functions flawlessly in Lucan. A duplicate PCI within 2 km of a macrocell is highly indicative of a rogue simulator.15

However, the exponential growth in mobile data consumption has forced operators to deploy Heterogeneous Networks (HetNets).50 In environments like the Dublin Docklands, local authorities and operators (e.g., Vodafone, Eir, Dense Air) have deployed extensive grids of small cells, picocells, and microcells attached to street furniture, traffic lights, and smart waste bins.49

These picocells have a maximum operational range of 100 to 250 meters.37 This densification drastically shrinks the necessary PCI reuse distance.15 To manage the finite pool of 504 identifiers across thousands of micro-nodes, operators are forced to reuse PCIs aggressively, often placing identical PCIs on different physical blocks within dense urban corridors.15

Therefore, a user commuting from Lucan to the Dublin Docklands will cross a threshold of densification. In the Docklands, the 2 km PCI\_COLLISION radius is computationally obsolete. Legitimate small cells located 800 meters apart will share a PCI, generating severe, continuous false positives.40

Furthermore, Self-Organizing Network (SON) algorithms operate autonomously.40 A small cell detecting local interference may automatically execute a SON optimization routine, swapping its PCI overnight to improve throughput.15 A UE observing this legitimate network adjustment will falsely interpret the event as a PCI\_INSTABILITY attack.15

## **6\. False Positive Cascades Induced by Infrastructure Mergers (MOCN)**

Beyond the physical densification of antennas, logical re-architecture poses an even more complex threat to the fidelity of IMSI catcher heuristics. The ongoing strategic network integration between major operators illustrates how corporate restructuring perfectly mimics the exact cryptographic signatures of a sophisticated cell-site simulator.

### **6.1 Multi-Operator Core Network (MOCN) Deployments**

In Ireland and the United Kingdom, intense market competition and the exorbitant costs of 5G deployment have driven rival carriers into strategic infrastructure joint ventures.57 Vodafone Ireland and Three Ireland have established agreements to pool their site portfolios, sharing physical network infrastructure across approximately 2000 macro locations.57

To share physical towers while strictly maintaining the independence of their respective core networks and subscriber billing systems, these operators implement Multi-Operator Core Network (MOCN) architecture.59 MOCN allows a single physical Radio Access Network (RAN)—the antenna arrays and the eNodeB baseband processing units—to serve subscribers from entirely distinct operators simultaneously.58

Under MOCN protocols, a single physical hardware sector (identified by its EUTRAN Cell ID) broadcasts multiple Public Land Mobile Network (PLMN) identifiers in its System Information Blocks.58 If a Vodafone user connects, the eNodeB routes the signaling traffic to the Vodafone core via the S1-MME interface. If a Three user connects to the exact same hardware, the traffic is routed to the Three core.58

### **6.2 The MOCN-Induced Heuristic Failure**

The deployment of MOCN networks completely shatters the foundational chronological and spatial assumptions built into basic IMSI catcher monitoring applications. The integration of MOCN directly triggers three distinct high and medium-severity alerts:

1. **Cell Renumbering and POPUP\_TOWER Alerts:** Consider a legacy Vodafone macrocell located in Lucan, historically broadcasting the PLMN 272-01.7 Under the MOCN integration, this physical hardware is reconfigured to simultaneously broadcast the PLMN for Three Ireland (272-05).8 To support the Three broadcast, the sector must broadcast a Three-compatible network identifier (e.g., 234-20-123456789).58 However, if that specific identifier already exists elsewhere in Three's legacy national network, routing conflicts will occur in the core. To resolve this, RF engineers must logically "renumber" the cell.58 From the perspective of the application's local database, an entirely new, unrecorded tower identity has instantaneously materialized within a fully mature 24-hour baseline. Despite the physical steel and silicon remaining entirely static, the application perceives a geographical impossibility, triggering a massive POPUP\_TOWER event.58  
2. **Logical Duplication and OPERATOR\_MISMATCH:** Forensic analysts and RF survey teams have documented that MOCN rollouts result in the sudden proliferation of "duplicate" cells.58 The detection app will record two distinct cell identities broadcasting from the exact same spatial coordinates, featuring identical Timing Advance and RSRP characteristics, yet presenting different MCC/MNC pairings.58 To a rigid heuristic algorithm, a single tower claiming to be both Vodafone and Three is the exact definition of an aggressive IMSI catcher attempting to overshadow multiple carriers simultaneously.  
3. **Routing Area Fluctuations and LAC\_TAC\_CHANGE:** MOCN agreements generally apply only to 4G LTE and 5G Non-Standalone (NSA) architectures.58 Legacy 2G networks (which Three Ireland does not operate) are excluded from the sharing architecture.58 As the UE dynamically balances loads across the MOCN hardware, the reported Tracking Area Code (TAC) can fluctuate rapidly as the device evaluates the virtualized layers.61 Furthermore, if a UE experiences slight signal degradation, it may legitimately fall back from the shared 4G MOCN layer to Vodafone's proprietary legacy 2G layer.58 This sequence of events will trigger a cascade of LAC\_TAC\_CHANGE alerts (due to the rapid TAC flips on the virtual layers), immediately followed by a high-severity DOWNGRADE\_2G alert. The app will interpret this legitimate infrastructure behavior as a highly coordinated, active cryptographic downgrade attack.

## **7\. Strategic Algorithmic Mitigation and Hardening**

Given the extreme volatility of biological signal attenuation, urban densification, and corporate network virtualization, mitigating false positives requires transitioning the heuristic model from evaluating *correlated symptoms* to demanding *causal confirmation* and contextual multi-dimensional validation.31

### **7.1 Hardening Spatial and Topographical Logic**

**Dynamic Optimization of the PCI Radius:** The static 2 km collision radius utilized by PCI\_COLLISION must be rendered dynamically aware of HetNet density.15 The application should autonomously calculate a "density index" by monitoring the total volume of unique CIDs detected per square kilometer. In sparse, macrocell-dominated regions (e.g., rural baselines), the 2 km limit is strictly maintained. However, when the density index crosses a threshold indicative of an urban microcell deployment, the collision radius algorithm must aggressively compress to 500 meters or less, accommodating the legitimate, mathematically necessary aggressive reuse of the 504 PCI pool.15

**Hexadecimal Baseband Parsing for Sibling Identification:** Android's CellIdentityLte.getCi() method returns a continuous long integer representing the full 28-bit E-UTRAN Cell Identifier.12 The internal architecture of this identifier is strictly partitioned: the top 20 bits define the eNodeB ID, and the bottom 8 bits designate the specific sector.12 To accurately suppress POPUP\_TOWER and IMPOSSIBLE\_MOVE anomalies resulting from cell renumbering, the detection algorithm must isolate and track the core eNodeB ID via modulo arithmetic and hexadecimal parsing. For example, a CID integer of 1046022 converts to the hexadecimal string FF606.13 By extracting FF6 (decimal 4086), the algorithm identifies the core hardware.13 If a new sector (06) materializes, but the eNodeB (FF6) is already highly trusted within the baseline, the algorithm can intelligently suppress the POPUP\_TOWER alert, recognizing the event as a minor sector reconfiguration rather than a rogue physical deployment.

**Temporal Smoothing for Biological Attenuation:** To counteract the false positives generated by SUSPICIOUS\_PROXIMITY due to human body shadowing, the RSRP thresholding parameters must integrate a temporal smoothing filter (low-pass filtering). Because a pedestrian turning their back to a small cell can instantly induce a 10 to 20 dB attenuation spike 43, instantaneous drops in RSRP while the Timing Advance remains at ![][image3] must be discarded. The algorithm should mandate that the RSRP remain persistently suppressed within the anomalous band for a continuous window (e.g., \> 60 seconds) to successfully filter out transient indoor multipath fading and physical obstruction events.38

### **7.2 Suppressing Infrastructure Virtualization (MOCN) Anomalies**

To neutralize the false positive storms generated by MOCN deployments, the application must correlate temporal and spatial anomalies across multiple PLMN boundaries simultaneously.58

If a POPUP\_TOWER event occurs concurrently across both the 272-01 (Vodafone Ireland) and 272-05 (Three Ireland) PLMNs, and both virtual towers share the exact same localized PCI, Timing Advance, and geographic signature, the application's logic engine can definitively classify the event as a multi-operator physical site upgrade.58 Clandestine IMSI catchers are historically constrained by hardware limits and rarely possess the transmission bandwidth or processing power required to seamlessly emulate a perfectly synchronized, multi-carrier MOCN handover environment.42 By identifying parallel PLMN anomalies, the engine safely whitelists the tower.

### **7.3 Multi-Dimensional Corroboration and the User-Space Boundary**

Ultimately, heuristics constructed atop the Android TelephonyManager operate at the periphery of the OSI model's higher communication layers.6 They remain blind to the cryptographic Layer 3 signaling where the attack mathematically occurs.6 Without root access to directly intercept the unauthenticated TAU Reject messages or the malicious Identity Requests from the baseband processor 6, non-root applications will inherently carry a residual margin of error.6

In the absence of raw baseband visibility, absolute precision is impossible. Therefore, the architectural philosophy of the detection engine must shift toward multi-dimensional corroboration.31 A solitary LAC\_TAC\_CHANGE or an isolated PCI\_COLLISION alert in a modern urban center is statistically meaningless.16 However, if a UE registers an IMPOSSIBLE\_MOVE, followed instantaneously by a DOWNGRADE\_2G event, and is subjected to a sustained, unyielding SIGNAL\_ANOMALY from a single eNodeB that fails hexadecimal sibling parsing—this exact synchronization of variables crosses the threshold of environmental coincidence. It provides the high-fidelity corroboration required to legitimately warn a user of a targeted, active cryptographic interception.

## **8\. Conclusion**

The detection of International Mobile Subscriber Identity catchers via device-level analytics represents a continuous, asymmetrical conflict between mobile security heuristics and the aggressive evolution of global telecommunications infrastructure. As demonstrated by the complex interplay of biological signal attenuation, physical cell identity limitations, and macro-architectural shifts like the Multi-Operator Core Network rollouts in urban environments, the radio frequency landscape is inherently chaotic.

Algorithms utilizing static, rigid geographical and temporal thresholds will inevitably degrade, drowning users in false positives as network operators relentlessly densify micro-cellular architecture and virtualize core routing functions. Optimizing these defense systems requires a fundamental transition from monitoring isolated heuristic triggers to demanding dynamic, multi-dimensional spatial modeling.

By dissecting hexadecimal baseband identifiers to monitor physical hardware rather than logical sectors, implementing dynamic collision radii that natively adapt to urban densification, and correlating anomalies across multiple operator codes to verify legitimate infrastructure sharing, detection platforms can mathematically filter the noise of modern network expansion. While deterministic identification remains restricted by the proprietary limitations of the baseband processor, rigorously contextualized algorithms ensure that mobile privacy applications remain a credible, highly effective defense against the pervasive threat of covert cellular surveillance.
